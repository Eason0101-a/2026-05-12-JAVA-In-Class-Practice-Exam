# AI Shooter Challenge

## 摘要

本專題以 Java Swing 實作 2D 塔防射擊遊戲，遊戲融合玩家操控、敵人 AI 尋路、障礙物碰撞檢測等核心機制。敵人採用廣度優先搜尋（BFS）演算法在格子地圖上動態尋找玩家，同時繞開障礙物並逐步逼近目標。遊戲支援分層關卡、暫停/重開、即時評分與 HP 系統，適合作為物件導向程式設計與資料結構課程的綜合實踐教材。

## 研究目標

1. 利用 BFS 演算法實作敵人的智慧追蹤與尋路機制，展示圖搜尋在遊戲 AI 中的應用。
2. 實現格子地圖碰撞檢測，確保敵人能動態避開靜態與動態障礙物。
3. 設計分層關卡系統，逐步增加敵人數量與難度。
4. 整合 Java Swing GUI，實現玩家輸入、動畫渲染、物理更新的即時互動。
5. 提供可擴充的遊戲框架，便於後續增加新敵人類型、道具、特效等功能。

## 系統設計

### 1. 核心類別架構

- **Main**：遊戲啟動入口
- **GameFrame**：頂層視窗容器
- **GamePanel**：遊戲邏輯與渲染主體
- **Player**：玩家角色，支援 WASD/方向鍵移動與射擊
- **Enemy**：敵人實體，透過 BFS 尋路追蹤玩家
- **Bullet**：玩家子彈
- **EnemyBullet**：敵人子彈
- **GridMap**：格子地圖與碰撞管理
- **PathFinder**：BFS 路徑搜尋引擎
- **Obstacle**：靜態障礙物
- **Direction**：方向列舉（上、下、左、右）
- **Node**：格子節點資料結構

### 2. BFS 路徑搜尋機制

敵人將遊戲區域視為 `width × height` 的格子網絡，每個格子可標記為可行走或阻擋。敵人在每次邏輯更新時：

1. 取得當前座標與玩家座標對應的格子
2. 呼叫 `PathFinder.findPath()` 以 BFS 執行搜尋
3. 取得路徑序列，每步只沿路徑向前移動一格
4. 若路徑被障礙物阻擋，重新搜尋更新

**時間複雜度**：$O(w \times h)$（單次 BFS）

**空間複雜度**：$O(w \times h)$（訪問集合與佇列）

### 3. 碰撞檢測與互動

- 敵人與玩家碰撞：玩家扣血
- 玩家子彈與敵人碰撞：敵人扣血、消滅、加分
- 角色與障礙物碰撞：阻擋移動
- 敵人子彈與玩家碰撞：玩家扣血

### 4. 遊戲狀態管理

- **遊戲進行**：玩家、敵人、子彈實時更新與渲染
- **暫停**：暫停所有更新，保留渲染
- **遊戲結束**：玩家 HP ≤ 0 或關卡全滅
- **關卡進度**：敵人全滅時進入下一關

## 操作流程

1. 啟動程式，遊戲視窗開啟，第 1 關開始
2. 使用 **WASD** 或 **方向鍵** 移動玩家
3. 按 **SPACE** 發射子彈
4. 摧毀所有敵人以進入下一關
5. 玩家 HP 歸零時遊戲結束，可按 **R** 重新開始
6. 按 **P** 暫停/恢復遊戲

## 時間複雜度分析

令：
- `w`、`h` = 遊戲區寬高（格子數）
- `n` = 敵人數量
- `m` = 子彈總數

### 每幀邏輯更新

**玩家更新**：$O(1)$

**敵人更新**：
- 單個敵人的 BFS 尋路：$O(w \times h)$
- 單個敵人的移動與射擊：$O(1)$
- 共 `n` 個敵人：$O(n \times w \times h)$

**子彈更新**：$O(m)$

**碰撞檢測**：
- 玩家子彈與敵人：$O(m \times n)$
- 敵人子彈與玩家：$O(m)$
- 整體：$O(m \times n)$

**每幀總複雜度**：$O(n \times w \times h + m \times n)$

### 最佳化

- BFS 結果可快取數幀，避免每幀都重算
- 使用空間分割（如四分樹）減少碰撞檢測次數

## 執行方式

### 需求

- Java 17 或以上
- 支援 Java Swing 的作業系統（Windows、macOS、Linux）

### 編譯

在專案根目錄執行下列指令（Windows PowerShell）：

```powershell
$files = Get-ChildItem -Recurse src/main/java -Filter *.java | ForEach-Object { $_.FullName }
javac -encoding UTF-8 -d out $files
```

### 執行

```powershell
java -cp out ai.shooter.Main
```

## 範例輸出

### 遊戲畫面特徵

1. **視窗標題**：顯示當前關卡、分數、玩家 HP
2. **玩家角色**：綠色矩形，位於視窗底部中央
3. **敵人**：紅色矩形，逐步逼近玩家
4. **障礙物**：灰色矩形，阻擋移動路徑
5. **子彈**：小型矩形，自玩家向上發射
6. **狀態欄**：顯示當前 HP、分數、關卡進度

### 遊戲進程示例

1. 關卡 1：5 個敵人，玩家存活消滅全部 → 進入關卡 2
2. 關卡 2：10 個敵人，難度提升
3. 若玩家 HP 歸零：遊戲結束，按 R 重新開始

## 注意事項

1. **敵人尋路**：BFS 計算較耗效能，若敵人過多可能影響幀率，可考慮快取路徑或增加搜尋間隔。
2. **解析度調整**：可修改 `GameFrame` 中的視窗大小常數調整遊戲解析度。
3. **難度平衡**：關卡敵人數量、移動速度、子彈傷害可在 `GamePanel` 中自訂。
4. **編碼設定**：編譯時須使用 `-encoding UTF-8` 確保中文註解正確顯示。
5. **首次執行**：Java JIT 編譯可能導致首次運行較慢，後續運行會更快。
6. **異常處理**：遊戲若發生異常會在控制台輸出，便於除錯。

## 結論

本實作透過 Java Swing 與 BFS 演算法綜合展示物件導向程式設計、資料結構與遊戲開發的整合應用。敵人的智慧尋路機制提供了圖搜尋演算法的實際應用案例，而分層關卡系統展示了遊戲難度遞進的設計思想。該實作適合作為高級 Java 課程的專案實踐，也可作為進階遊戲開發的基礎框架，便於後續擴展敵人 AI、道具系統、特殊關卡等功能。
